# 4.3 로깅: Loki + Fluent Bit — 예상 결과

## 상태 확인

```
$ kubectl get pods -n monitoring -l app.kubernetes.io/name=loki
NAME     READY   STATUS    RESTARTS   AGE
loki-0   1/1     Running   0          2m

$ kubectl get pods -n monitoring -l app.kubernetes.io/name=fluent-bit
NAME               READY   STATUS    RESTARTS   AGE
fluent-bit-xxxxx   1/1     Running   0          2m
fluent-bit-xxxxx   1/1     Running   0          2m
```

## 체크리스트

- [ ] Loki SingleBinary 모드 Running
- [ ] Fluent Bit DaemonSet Running (노드당 1개)
- [ ] Grafana에서 Loki 데이터소스 추가됨
- [ ] `{namespace="notiflex"}` 쿼리로 로그 조회 가능

## 이 단계 이후 아키텍처

```
GKE: notiflex-cluster
├── monitoring namespace
│   ├── Prometheus + Grafana + Alertmanager
│   ├── Loki (로그 저장)
│   └── Fluent Bit (로그 수집, DaemonSet)
├── argocd namespace
└── notiflex namespace
```
