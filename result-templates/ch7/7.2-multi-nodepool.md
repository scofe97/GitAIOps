# 7.2 멀티 노드풀 — 예상 결과

## 상태 확인

```
$ kubectl get nodes
NAME                                              STATUS   ROLES    AGE
gke-sysnet4admin_book_gitaiops-cluster-api-pool-xxxxxxxx-xxxx       Ready    <none>   2m
gke-sysnet4admin_book_gitaiops-cluster-default-pool-xxxxxxxx-xxxx   Ready    <none>   1h
gke-sysnet4admin_book_gitaiops-cluster-default-pool-xxxxxxxx-xxxx   Ready    <none>   1h
gke-sysnet4admin_book_gitaiops-cluster-ops-pool-xxxxxxxx-xxxx       Ready    <none>   2m
gke-sysnet4admin_book_gitaiops-cluster-worker-pool-xxxxxxxx-xxxx    Ready    <none>   2m

$ kubectl get pods -n notiflex -o wide
NAME                            READY   NODE
notiflex-api-xxx-xxx            1/1     gke-...-api-pool-xxx
notiflex-api-xxx-xxx            1/1     gke-...-api-pool-xxx
```

## 체크리스트

- [ ] api-pool (e2-medium), worker-pool (e2-standard-2), ops-pool (e2-small) 생성됨
- [ ] 모두 Spot VM, 노드 1개씩
- [ ] Notiflex Pod이 api-pool에 배치됨 (nodeSelector)
- [ ] vCPU 쿼터 초과하지 않음

## 이 단계 이후 아키텍처

```
GKE: notiflex-cluster (5 nodes)
├── default-pool: e2-medium x2 (시스템)
├── api-pool: e2-medium x1 (Notiflex API)
├── worker-pool: e2-standard-2 x1 (Kafka 등)
└── ops-pool: e2-small x1 (운영 도구)

노드 배치:
  api-pool     → notiflex-api
  worker-pool  → (8장에서 Kafka 배치)
  ops-pool     → (8장에서 CronJob 배치)
```
