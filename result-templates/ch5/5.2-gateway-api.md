# 5.2 Gateway API 도입 — 예상 결과

## 상태 확인

```
$ kubectl get gateway -n notiflex
NAME                CLASS                              ADDRESS          PROGRAMMED   AGE
notiflex-gateway    gke-l7-regional-external-managed   35.xxx.xxx.xxx   True         3m

$ curl http://35.xxx.xxx.xxx/health
{"status":"ok"}

$ curl http://35.xxx.xxx.xxx/id
{"id":"1","generated_by":"notiflex-api-xxxxxxxxxx-xxxxx"}
```

## 체크리스트

- [ ] Gateway에 외부 IP 할당됨
- [ ] HTTPRoute 생성됨
- [ ] HealthCheckPolicy 설정됨 (/health:8080)
- [ ] 외부에서 /health, /id 정상 응답

## 이 단계 이후 아키텍처

```
[인터넷] → [Gateway API (35.xxx.xxx.xxx)]
                    ↓ HTTPRoute
           [notiflex-api Service]
                    ↓
           [notiflex-api Pod x2]

GKE: notiflex-cluster
├── notiflex namespace
│   ├── Gateway: notiflex-gateway (외부 IP)
│   ├── HTTPRoute: / → notiflex-api
│   ├── HealthCheckPolicy: /health:8080
│   └── notiflex-api (v0.1.x)
├── monitoring namespace
└── argocd namespace
```
