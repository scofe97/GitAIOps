# 8.3 배치 자동화: CronJob

## 실행 지침

### 단계 1: 헬스체크 CronJob 생성

`k8s/smb/healthcheck-cronjob.yaml` 파일을 생성한다:

- **스케줄**: `*/5 * * * *` (5분마다)
- **이미지**: `curlimages/curl:latest`
- **노드 배치**: ops-pool (`nodeSelector: cloud.google.com/gke-nodepool: ops-pool`)
- **동작**: `http://notiflex-api.notiflex.svc.cluster.local/health` 호출
  - 200 → 성공 로그
  - 그 외 → 실패 (exit 1)
- **리소스**: requests 10m/16Mi, limits 50m/32Mi
- **restartPolicy**: OnFailure
- **히스토리**: successfulJobsHistoryLimit: 3, failedJobsHistoryLimit: 3

### 단계 2: 적용 및 확인

```bash
kubectl apply -f k8s/smb/healthcheck-cronjob.yaml
kubectl get cronjob -n notiflex
```

5분 후 Job이 생성되는지 확인:
```bash
kubectl get jobs -n notiflex
kubectl logs job/<job-name> -n notiflex
```

### 단계 3: JOURNEY.md 도구 선택 기록 추가

이 장의 결정(배치 자동화 — K8s CronJob)은 ch9 회고와 ADR-016으로 누적되므로 `JOURNEY.md`의 **도구 선택 기록 테이블에 한 줄을 추가**한다. 비워두면 `/update-docs` 시점에 ADR로 변환되지 못해 누락된다.

```
| 배치 자동화 (ch8.3) | K8s CronJob | 외부 cron + 쿠버네티스 외부 트리거, Argo Workflows | 쿠버네티스 네이티브, ops-pool 배치, ArgoCD가 매니페스트로 관리 |
```

> ⚠️ "도구 선택"이라는 단어 때문에 결정 항목을 빠뜨리기 쉽다. CronJob처럼 "도구 비교"가 아니라 "스케줄링 결정"인 항목도 동일하게 추가한다. ADR-016은 이 행을 source로 만들어진다.

### 단계 4: 커밋

변경 사항을 커밋하고 푸시한다.

## 트러블슈팅

### ArgoCD가 CronJob을 즉시 인식하지 못함
→ 새로운 매니페스트 파일(healthcheck-cronjob.yaml)을 추가하고 git push 후 ArgoCD sync를 실행해도, CronJob이 ArgoCD Application에 즉시 나타나지 않을 수 있다.
→ ArgoCD의 파일 감지 주기(기본 3분) 때문이다. `argocd app sync notiflex-smb`가 "already synced"를 반환하면서 새 파일을 반영하지 않는 경우가 있다.
→ 해결: `kubectl apply -f k8s/smb/healthcheck-cronjob.yaml -n notiflex`로 직접 적용한다. ArgoCD는 이후 자동으로 인식한다.
→ 또는 ArgoCD UI에서 Hard Refresh 후 Sync를 실행한다.
