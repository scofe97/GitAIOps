# 2.3 gcloud CLI 설치

## 실행 지침

### 단계 1: gcloud CLI 설치 확인

gcloud CLI가 설치되어 있는지 확인한다:
```bash
gcloud version
```

설치되어 있지 않으면 OS에 맞는 설치 방법을 안내한다.

### 단계 2: GCP 인증

```bash
gcloud auth login
gcloud auth application-default login
```

> ⚠️ 이 명령은 브라우저 인증이 필요하므로 독자가 직접 실행해야 한다. Claude Code에서는 `! gcloud auth login` 형태로 실행을 안내한다.

### 단계 3: 프로젝트 설정

프로젝트 ID와 기본 존을 설정한다. 이 설정이 없으면 이후 모든 gcloud 명령에서 `--project`, `--zone` 플래그를 매번 지정해야 한다.

```bash
# 현재 설정 확인
gcloud config list

# 프로젝트 설정 (GCP 콘솔 > 프로젝트 선택기에서 프로젝트 ID 확인)
gcloud config set project <PROJECT_ID>

# 기본 존 설정 (이후 클러스터 생성 시 --zone 생략 가능)
gcloud config set compute/zone asia-northeast3-a

# 기본 리전 설정 (Artifact Registry, Cloud Build 등 region 단위 서비스에서 --location 생략 가능)
gcloud config set compute/region asia-northeast3
```

> ⚠️ `gcloud config set project`를 하지 않으면 클러스터 생성, 이미지 빌드 등 모든 명령에서 `--project` 플래그가 필요하다. 반드시 먼저 설정한다.

### 단계 4: Artifact Registry Docker 인증

Artifact Registry에 컨테이너 이미지를 푸시하려면 Docker 인증이 필요하다:

```bash
gcloud auth configure-docker asia-northeast3-docker.pkg.dev
```

> ⚠️ 이 명령을 실행하지 않으면 2.6에서 `gcloud builds submit`은 정상 동작하지만, 로컬에서 `docker push`로 직접 푸시할 때 `unauthorized` 에러가 발생한다.
