# 2.6 Notiflex 앱 생성·빌드·배포

## 실행 지침

### 단계 1: Notiflex API 서버 생성

`app/main.go`를 생성한다:
- Go 표준 라이브러리만 사용 (net/http)
- 엔드포인트: `GET /health` (상태 확인), `GET /id` (고유 ID 생성)
- `/id`는 인메모리 카운터로 순차 ID 생성, 생성한 Pod 이름을 함께 반환
- 포트: 8080

`app/Dockerfile`을 생성한다:
- 멀티스테이지 빌드: `golang:1.25-alpine` → scratch
- CGO_ENABLED=0 으로 정적 바이너리

`app/go.mod`을 생성한다:
- `go mod init notiflex` 실행
- Go 버전은 `1.25`로 설정 (이후 ch6에서 추가하는 valkey-go는 Go 1.24+, ch8에서 추가하는 OTel SDK는 Go 1.25+를 요구하므로, 처음부터 1.25로 시작하면 이후 의존성 추가 시 버전 변동이 없다)

> ⚠️ **Go 버전을 1.25로 통일**: 이 프로젝트는 최종적으로 Go 1.25 이상이 필요한 의존성(OTel SDK v1.42+)을 사용한다. 처음부터 `golang:1.25-alpine` + `go 1.25`로 시작하면 이후 챕터에서 Go 버전 불일치 문제가 발생하지 않는다.

### 단계 2: Artifact Registry에 이미지 빌드·푸시

```bash
# Artifact Registry 리포 생성 (없으면)
gcloud artifacts repositories create notiflex --repository-format=docker --location=asia-northeast3

# 빌드 및 푸시
gcloud builds submit app/ --tag=asia-northeast3-docker.pkg.dev/<PROJECT_ID>/notiflex/api:v0.1.0
```

### 단계 3: K8s 매니페스트 생성 및 배포

`k8s/smb/` 디렉터리에 생성:
- `namespace.yaml`: notiflex 네임스페이스
- `deployment.yaml`: replicas 2, 이미지 v0.1.0, readiness/liveness probe 포함
- `service.yaml`: ClusterIP, port 80 → targetPort 8080

```bash
# 네임스페이스를 먼저 생성한 후 나머지 리소스를 적용한다
kubectl apply -f k8s/smb/namespace.yaml
kubectl apply -f k8s/smb/
```

> ⚠️ **Namespace race condition**: `kubectl apply -f k8s/smb/`로 한번에 적용하면 네임스페이스가 아직 생성되지 않은 상태에서 Deployment/Service가 적용되어 `namespace not found` 에러가 발생할 수 있다. 네임스페이스를 먼저 적용한다.

### 단계 4: 배포 확인

```bash
kubectl get pods -n notiflex
kubectl port-forward svc/notiflex-api -n notiflex 8080:80 &
curl http://localhost:8080/health
curl http://localhost:8080/id
```
