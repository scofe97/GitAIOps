# 2.4 GitHub 저장소 구성

## 실행 지침

### 단계 1: 책 저장소 clone

_Book_GitAIOps 저장소를 clone한다 (이미 되어 있으면 건너뜀).

### 단계 2: notiflex-platform 저장소 생성

이 저장소(_Book_GitAIOps)의 **형제 디렉터리**에 생성한다. 반드시 이 저장소의 부모 디렉터리로 이동한 후 실행한다:
```bash
cd "$(git rev-parse --show-toplevel)/.."
gh repo create notiflex-platform --private --clone
cd notiflex-platform
```

> ⚠️ **저장소가 이미 존재하는 경우** (재실습 시): 기존 파일을 모두 삭제하는 커밋을 만들고 force-push로 리셋한다. `rm -rf`로 디렉터리 자체를 삭제하지 않는다.
> ```bash
> cd ../notiflex-platform
> git rm -rf . 2>/dev/null; git clean -fd
> git commit --allow-empty -m "init"
> git push origin main --force
> ```

### 단계 3: CLAUDE.md 생성

notiflex-platform 저장소 루트에 CLAUDE.md를 생성한다. 포함 내용:
- **프로젝트 개요**: Notiflex — B2B 알림 SaaS 플랫폼
- **기술 스택**: Go 표준 라이브러리, scratch 베이스 컨테이너
- **GCP 설정**: 현재 gcloud 설정에서 프로젝트 ID, 리전, 존을 읽어서 기록
- **Artifact Registry**: `asia-northeast3-docker.pkg.dev/<PROJECT_ID>/notiflex`
- **행동 규칙**: 항상 확인 후 실행, 변경 전 현재 상태 확인

### 단계 4: 디렉터리 구조 생성

```
notiflex-platform/
├── CLAUDE.md
├── app/           # Go 애플리케이션
├── k8s/
│   └── smb/       # K8s 매니페스트
└── .github/
    └── workflows/ # CI 파이프라인
```
